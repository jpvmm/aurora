# Phase 04.1 Plan 02: Memory-First Retrieval, Three-Way Routing, and Dual-Source Ask Summary

**One-liner:** Memory-first retrieval via `retrieve_memory_first()`, three-way ChatSession routing (vault/memory/chat), and dual-source `aurora ask` using vault + memory collections.

## What Was Built

### Task 1: `retrieve_memory_first` + Three-Way ChatSession Routing

**RetrievalService** (`src/aurora/retrieval/service.py`):
- Added `retrieve_memory_first(query)` — mirrors `retrieve_with_memory` but places memory notes before vault notes in `result.notes`. Memory backend failures are treated as empty results (Pitfall 3 safety). Combined context respects `MAX_CONTEXT_CHARS`.

**ChatSession** (`src/aurora/chat/session.py`):
- Imported `get_system_prompt_memory_first` from `aurora.llm.prompts`
- Updated `process_turn()` from two-way (vault/chat) to three-way routing: vault → `_handle_vault_turn`, memory → `_handle_memory_turn`, else → `_handle_chat_turn`
- Added `_handle_memory_turn()`: calls `retrieve_memory_first`, uses `get_system_prompt_memory_first`, emits "Buscando nas memorias..." status
- Updated class docstring to document all three intent paths

### Task 2: Dual-Source `aurora ask`

**aurora ask CLI** (`src/aurora/cli/ask.py`):
- Imports `QMDSearchBackend`, `MEMORY_COLLECTION`, `MEMORY_INDEX`
- Creates `memory_backend = QMDSearchBackend(index_name=MEMORY_INDEX, collection_name=MEMORY_COLLECTION)`
- Changed from `retrieval.retrieve(query)` to `retrieval.retrieve_with_memory(query)`
- Status message updated: `"Buscando no vault e memorias..."`
- Found-notes message now distinguishes vault vs memory counts when both present
- When memory notes present: uses `llm.chat_turn` with `get_system_prompt_grounded_with_memory` for dual-source grounding
- When vault-only results: continues using `llm.ask_grounded` (backward compatible)
- JSON output `sources` field now includes memory paths (e.g., `memory/2024-01.md`)
