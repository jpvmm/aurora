# Phase 04.1: Fix Memory Pipeline - Context

**Gathered:** 2026-04-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Fix the memory pipeline so it works end-to-end: three-way intent classification (vault/memory/chat), memory-first retrieval for memory questions, aurora ask also reads memories, improved summary quality for better retrieval, and clear memory-vault distinction in responses.

</domain>

<decisions>
## Implementation Decisions

### Three-Way Intent Classification
- **D-01:** Upgrade intent classifier from two-way (vault/chat) to three-way: vault / memory / chat.
  - `vault`: questions about notes, documents, vault content → search KB first, memory as supplement
  - `memory`: questions about past conversations, what was discussed, session history → search memory first, vault as supplement
  - `chat`: general conversation, no retrieval needed
- **D-02:** Update `INTENT_PROMPT` with three categories, clear examples for each, especially memory-related patterns ("o que conversamos", "lembra quando", "na ultima sessao").
- **D-03:** Update `LLMService.classify_intent()` to return "vault", "memory", or "chat" (instead of only "vault"/"chat").

### Memory-First Retrieval for Memory Intent
- **D-04:** When intent is `memory`: search aurora-memory collection first (priority), then search vault as supplement. Memory notes listed first in context.
- **D-05:** When intent is `vault`: search vault first (priority), then search memory as supplement. Vault notes listed first in context (existing behavior from Phase 4).
- **D-06:** Add `retrieve_memory_first()` method to RetrievalService — mirrors `retrieve_with_memory()` but reverses priority order (memory notes first, vault notes second).

### Aurora Ask Also Reads Memories
- **D-07:** Change `aurora ask` from vault-only to dual-source (vault + memory). Reuse the same `retrieve_with_memory()` pipeline.
- **D-08:** `aurora ask` status messages updated: "Buscando no vault e memorias..."
- **D-09:** Memory citations `[memoria: titulo]` appear in `aurora ask` output alongside vault citations.

### Memory-Vault Distinction in Responses
- **D-10:** System prompt explicitly instructs the model to label sources clearly: vault citations as `[notas/path.md]`, memory citations as `[memoria: titulo da sessao]`.
- **D-11:** When responding to memory-intent questions, system prompt emphasizes temporal context: "O usuario esta perguntando sobre conversas anteriores. Priorize as memorias e cite a data da sessao."

### Memory Summary Quality
- **D-12:** Upgrade summary format from free-form prose to structured markdown with sections: `## Topicos`, `## Decisoes`, `## Contexto`.
- **D-13:** Include date and time explicitly in the summary body text (not just frontmatter). Line: `Data da sessao: 2026-04-05`. Makes temporal queries ("conversamos hoje") matchable by QMD.
- **D-14:** Update `SUMMARIZE_SESSION_PROMPT` to require the structured format with date in body.

### ChatSession Routing Update
- **D-15:** `ChatSession.process_turn()` routes to three handlers:
  - `_handle_vault_turn()` — existing, uses `retrieve_with_memory()` (vault-first)
  - `_handle_memory_turn()` — new, uses `retrieve_memory_first()` (memory-first)
  - `_handle_chat_turn()` — existing, free-form

### Claude's Discretion

- Memory-first system prompt wording details
- How many memory results vs vault results in the combined context budget split

</decisions>
