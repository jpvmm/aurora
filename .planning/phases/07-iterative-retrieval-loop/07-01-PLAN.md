    `judge_sufficiency_deterministic(result, *, min_top_score, min_hits, min_context_chars, hybrid_top_score=None) -> SufficiencyVerdict` is a module-level pure function in `src/aurora/retrieval/service.py`. NO I/O, NO LLM, NO instance state. Lives next to `_extract_proper_nouns`.

    Signal ordering (D-02, RESEARCH "Sufficiency Signals — Detailed"):
    1. If `result.insufficient_evidence` is True OR `len(result.notes) == 0`: return `SufficiencyVerdict(ok=False, reason="sem resultados", signals=("hits=0",))`. Short-circuit, do not evaluate other signals.
    2. Compute the three signals (always all three, collect every trip into signals tuple — D-02 says "any one trips thin", but RESEARCH says "Collect all tripped signals into SufficiencyVerdict.signals for the trace and reformulation prompt"):
       - **Top-score check (mode-aware, RESEARCH finding 3):**

           result: RetrievalResult,
           *,
           min_top_score: float,
           min_hits: int,
           min_context_chars: int,
           hybrid_top_score: float | None = None,
       ) -> SufficiencyVerdict:

           Signal ordering (D-02): empty -> always thin; otherwise OR over
           top-score (mode-aware), hits-above-floor count, and context length.

           hybrid_top_score: caller passes the max score among HYBRID-origin hits
           only (RESEARCH finding 3 — BM25 scores have a different scale and would
           skew this check). Pass None to skip the top-score signal entirely.
           """

               return SufficiencyVerdict(

               )

           if hybrid_top_score is not None and hybrid_top_score < min_top_score:

               )
