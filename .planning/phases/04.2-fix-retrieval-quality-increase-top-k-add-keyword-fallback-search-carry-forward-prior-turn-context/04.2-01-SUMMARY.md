## Decisions Made

- `keyword_search()` mirrors `search()` implementation exactly, using `"search"` subcommand (BM25) vs `"query"` (hybrid). Same JSON parsing, same error handling — reduces cognitive overhead.
- Proper noun heuristic: skip first word (sentence-start bias), detect uppercase words (len >= 2) and double-quoted phrases. Simple and correct for Portuguese and English queries.
- Keyword fallback is additive: hybrid results always included if ok, keyword results appended, then deduplication keeps highest score per path.
- `_keyword_fallback()` silently swallows failed keyword searches (returns empty tuple). The hybrid result path is never blocked by a keyword search failure.
