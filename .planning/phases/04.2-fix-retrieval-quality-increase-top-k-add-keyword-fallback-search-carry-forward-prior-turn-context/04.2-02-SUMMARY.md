## Decisions Made
- Apply carry-forward BEFORE the `insufficient_evidence` check so prior notes can rescue a follow-up query that returns no fresh results on its own.
- Cap both stored paths (`_last_retrieved_paths`) and applied supplements at 3 to bound context growth.
- Clear carry-forward on chat-intent turns (D-11) to prevent prior vault notes leaking into unrelated conversational turns.
- Skip silently when `_backend.fetch()` returns None; no exception, no partial state mutation.
